# V2H-Rectify — LLM Agent Codebase Reference
# Version: 3.1.0
# Language: Traditional Chinese Handwritten OCR System
# Updated: 2026-03-22

## SYSTEM IDENTITY

V2H-Rectify (Vertical-to-Horizontal Rectify) is a full-stack OCR system that detects, corrects perspective, and transcribes Traditional Chinese handwritten text from scanned manuscript paper (稿紙). It converts vertical (top-to-bottom, right-to-left) handwriting into horizontal digital text.

## ARCHITECTURE

### Service Topology

```
frontend (nginx:80 → host:3000)
  ├── serves React SPA static files
  ├── proxies /api/* → backend:8000
  └── upgrades /api/v1/ws/* to WebSocket → backend:8000

backend (uvicorn:8000 → host:8080)
  ├── FastAPI with CRAFT text detection (GPU)
  ├── OpenCV image processing pipeline
  └── calls VLM via OpenAI-compatible API

vllm (vllm:8000 → host:8001)
  └── OpenAI-compatible inference server (Qwen2.5-VL)
```

### Request Flow

```
User uploads image
  → POST /api/v1/corner/detect        (returns corners + task_id)
  → POST /api/v1/corner/correct       (perspective warp, stores corrected image)
  → POST /api/v1/ocr/upload           (starts async OCR, returns task_id)
  → WS /api/v1/ws/{task_id}           (real-time progress: stage + progress + status)
  → GET /api/v1/ocr/{task_id}         (poll for result; triggered immediately on WS completion)
  → Response: { title, text, columns[], rotation_angle, elapsed_seconds }
```

## BACKEND MODULES

### api/app.py
- **Exports**: `app` (FastAPI instance), `create_app()` factory
- **Lifespan**: `async def lifespan(app)` — initializes CRAFTService, VLMService, OCRPipeline on startup; calls `craft_svc.stop()` + `clear_services()` on shutdown
- **Middleware**: CORS with `allow_origins=["*"]`
- **Route prefix**: `/api/v1`

### api/deps.py
- **Exports**: `get_craft()`, `get_vlm()`, `get_pipeline()` — FastAPI `Depends()` callables
- **Pattern**: Module-level singletons set by `set_services()` during lifespan, cleared by `clear_services()`

### api/schemas.py
- **Exports** (all Pydantic BaseModel):
  - `TaskStatus` (enum: pending, processing, completed, failed)
  - `Point` (x: float, y: float)
  - `CornerDetectResponse` (task_id, corners: List[Point], confidence)
  - `CornerCorrectRequest` (task_id, corners: List[Point])
  - `CornerCorrectResponse` (corrected: bool, preview_url)
  - `OCROptions` (auto_rotate, remove_print, auto_split — all bool, default True)
  - `OCRSubmitResponse` (task_id, status)
  - `ColumnData` (col_index, text, spacing_indexes: List[int], num_rows)
  - `OCRStatusResponse` (task_id, status, progress, title?, text?, columns[], rotation_angle?, elapsed_seconds?, error?)
  - `HealthResponse` (status, craft_loaded, vlm_backend, version)
  - `WSProgressMessage` (task_id, stage, progress, message, status)  ← status field added

### api/ws.py
- **Exports**: `router` (APIRouter), `broadcast_progress(task_id, stage, progress, message, status)`
- **Endpoint**: `WS /ws/{task_id}` — accepts connection, adds to `_connections` dict, keeps alive until disconnect
- **Broadcast**: sends `WSProgressMessage` JSON to all connected clients for a task_id
- **Status**: now includes `status` field so clients know when task is completed/failed via WS

### api/routes/corner.py
- **Prefix**: `/corner`
- **In-memory store**: `_pending_images: dict[str, np.ndarray]` — maps task_id to BGR image
- `POST /detect`: Upload image → `detect_corners(img)` → returns corners + task_id. Falls back to full image corners with confidence=0 if detection fails or area < 0.35
- `POST /correct`: Takes `CornerCorrectRequest` → `warp_perspective(img, corners)` → stores warped image back in `_pending_images`

### api/routes/ocr.py
- **Prefix**: `/ocr`
- **In-memory store**: `_tasks: Dict[str, dict]`
- `POST /upload`: Runs `_run_ocr()` in executor (non-blocking). Returns task_id + status=processing
- `GET /{task_id}`: Returns `OCRStatusResponse`
- **`_run_ocr(task_id, img_bgr, pipeline, opts, loop)`**:
  - Receives the running event loop as argument
  - Defines `on_progress(stage, progress)` callback that uses `asyncio.run_coroutine_threadsafe()` to safely call async `broadcast_progress()` from the sync executor thread
  - Passes `on_progress` to `pipeline.run()`
  - On completion: broadcasts `status="completed"` with progress=1.0

### api/routes/health.py
- `GET /health`: Returns `HealthResponse` with craft_loaded and vlm_backend status

### config/settings.py
- **Root class**: `Settings(BaseSettings)` with `env_nested_delimiter="__"`
- **Sub-configs**: `CRAFTSettings`, `VLMSettings`, `OCRPipelineSettings`, `ServerSettings`
- **Singleton**: `get_settings()` returns cached `Settings` instance
- **OCRPipelineSettings reformat fields** (4 total, down from 18):
  - `reformat_spacing: int = 20`
  - `reformat_binary_threshold: int = 128`
  - `reformat_blank_ink_ratio: float = 0.04`
  - `reformat_heat_rescue_peak: float = 0.40`

### services/craft_service.py
- **Class**: `CRAFTService`
- **Pattern**: Background worker thread + Queue for batched inference
- **Key methods**: `start()` / `stop()`, `detect(image) → CRAFTResult`
- **Thread safety**: Uses `queue.Queue` and `threading.Event`

### services/vlm_service.py
- **Class**: `VLMService`
- **Pattern**: Strategy pattern with 3 backends
- **Factory**: `VLMService.from_settings(vlm_settings)`
- **Key method**: `recognize(image_b64: str, prompt: str) → str`

### services/ocr_pipeline.py
- **Class**: `OCRPipeline`
- **Key method**: `run(pil_img, auto_rotate, remove_print, auto_split, on_progress=None) → OCRResult`
  - `on_progress: Callable[[str, float], None] | None` — called at each stage with (stage_name, progress_0_to_1)
- **Progress events emitted**:

  | Progress | Stage name (zh) |
  |---|---|
  | 0.10 | `"偵測文字區塊"` |
  | 0.25 | `"校正傾斜"` |
  | 0.40 | `"移除印刷文字"` |
  | 0.55 | `"提取格線結構"` |
  | 0.65 | `"重新格式化版面"` |
  | 0.65–0.95 | `"辨識第 N 欄"` (per-column, evenly distributed) |
  | 1.00 | `"完成"` |

- **Return type**: `OCRResult` dataclass (title, text, columns, rotation_angle, elapsed_seconds)

### core/perspective.py
- **Exports**: `Corners`, `detect_corners(img_bgr) → Corners | None`, `warp_perspective(img_bgr, corners) → np.ndarray`

### core/craft_detector.py
- **Exports**: `CRAFTResult` (immutable dataclass: boxes, confidences, image_shape), `run_craft(model, image)`

### core/grid_extractor.py
- **Exports**: `extract_grid(boxes, img_shape) → GridResult`

### core/rotation.py
- **Exports**: `estimate_angle(boxes) → float`, `correct_rotation(img, boxes) → tuple[np.ndarray, float]`

### core/print_removal.py
- **Exports**: `classify_print_handwritten(boxes)`, `remove_print_region(img, boxes) → np.ndarray`

### core/text_reformat.py
- **Exports**: `reformat_columns(img, grid_boxes, *, config, ink_map) → list[ReformattedColumn]`, `refine_text_raw(recognized_texts)`, `TextReformatConfig`
- **TextReformatConfig** (frozen dataclass, 4 fields):
  ```python
  spacing: int = 20
  binary_threshold: int = 128
  blank_ink_ratio: float = 0.04   # fill ratio below this = sparse ink
  heat_rescue_peak: float = 0.40  # heatmap peak above this = definitely a character
  ```
- **Module-level constants** (hardcoded, not configurable):
  - `_LINE_THICKNESS_RATIO = 0.10`
  - `_LINE_SPAN_RATIO = 0.50`
  - `_HEAT_ACTIVE_THRESHOLD = 0.35`
  - `_HEAT_RESCUE_ACTIVE = 0.03`
- **`_is_blank_cell(binary, ink_crop, bw, bh, cfg) → bool`** — priority decision tree:
  1. `heat_peak >= cfg.heat_rescue_peak` → NOT blank (strong character signal, highest priority)
  2. `fill_ratio == 0` → blank
  3. Line shape (thin horizontal/vertical) → if `heat_active >= _HEAT_RESCUE_ACTIVE`: NOT blank (e.g. 「一」); else: blank (printed grid line)
  4. `fill_ratio <= cfg.blank_ink_ratio` → if has heatmap: blank (noise); else: NOT blank (benefit of doubt)
  5. Meaningful ink → NOT blank
- **Key design**: line detection rescue is independent of global rescue — printed grid lines (heat_active < 3%) are never rescued

### core/image_utils.py
- **Exports**: `pil_to_bgr()`, `bgr_to_pil()`, `split_wide_image()`

## API ENDPOINTS — FULL SIGNATURES

### POST /api/v1/corner/detect
- **Request**: `multipart/form-data` with `file: UploadFile`
- **Response 200**: `{ task_id: str, corners: [{x,y}×4], confidence: float, preview_url: str }`
- **Corner order**: [top-left, top-right, bottom-right, bottom-left]
- **Side effect**: Stores BGR image in `_pending_images[task_id]`

### POST /api/v1/corner/correct
- **Request**: `{ task_id: str, corners: [{x,y}×4] }`
- **Response 200**: `{ corrected: true, preview_url: "" }`
- **Side effect**: Replaces `_pending_images[task_id]` with warped image

### POST /api/v1/ocr/upload
- **Request**: `multipart/form-data`: `file`, `auto_rotate`, `remove_print`, `auto_split`, `task_id?`
- **Response 200**: `{ task_id: str, status: "processing" }`
- **Side effect**: Starts background OCR, `on_progress` callback bridges to WebSocket via `run_coroutine_threadsafe`

### GET /api/v1/ocr/{task_id}
- **Response 200**: `{ task_id, status, progress(0-1), title?, text?, columns[], rotation_angle?, elapsed_seconds?, error? }`
- **Status transitions**: pending → processing → completed | failed

### WS /api/v1/ws/{task_id}
- **Protocol**: WebSocket
- **Server→Client messages**: `{ task_id, stage, progress(0-1), message, status }`
- **Active**: pipeline broadcasts at each processing stage; sends `status="completed"/"failed"` on finish
- **Frontend behavior**: on receiving completed/failed status, immediately fires a final REST poll (no 2s wait)

### GET /api/v1/health
- **Response 200**: `{ status: "ok", craft_loaded: bool, vlm_backend: str, version: "3.1.0" }`

## DATA FLOW — END-TO-END OCR PIPELINE

```
Input: PIL.Image
  │
  ├─[1] split_wide_image()           # if aspect_ratio > 2.0 and auto_split=True
  │
  ├─[2] craft_svc.detect(img)        # CRAFT text detection → on_progress("偵測文字區塊", 0.10)
  │     → CRAFTResult { boxes, confidences, image_shape }
  │
  ├─[3] correct_rotation(img, boxes) # if auto_rotate → on_progress("校正傾斜", 0.25)
  │
  ├─[4] remove_print_region(img)     # if remove_print → on_progress("移除印刷文字", 0.40)
  │
  ├─[5] extract_grid(boxes)          # → on_progress("提取格線結構", 0.55)
  │
  ├─[6] reformat_columns(img, grid)  # → on_progress("重新格式化版面", 0.65)
  │     Uses TextReformatConfig (4 params) + _is_blank_cell() decision tree
  │
  ├─[7] CONCURRENT VLM OCR           # ThreadPoolExecutor(max_workers=4)
  │     for each column:             # → on_progress("辨識第 N 欄", 0.65 + N/total * 0.30)
  │       → vlm_svc.recognize(image_b64, prompt) → text
  │
  └─[8] refine_text_raw() + title detection → on_progress("完成", 1.00)

Output: OCRResult { title, text, columns[], rotation_angle, elapsed_seconds }
```

## CONVENTIONS & PATTERNS

### Backend Patterns
- **Dependency Injection**: FastAPI `Depends()` with singleton getters in `deps.py`
- **Lifespan Management**: `@asynccontextmanager async def lifespan(app)`
- **Strategy Pattern**: `VLMService` with pluggable backends
- **Batching Pattern**: `CRAFTService` uses Queue + worker thread
- **Executor Bridge**: `run_coroutine_threadsafe(loop, coro)` to call async from sync executor thread
- **Ref-mirrors-State**: `viewRef` in ZoomableImage mirrors React state for non-React event handlers
- **Non-passive Wheel**: `el.addEventListener("wheel", fn, { passive: false })` via useEffect for zoom-without-scroll
- **In-memory state**: Both `_pending_images` and `_tasks` are plain dicts (not Redis)

### Frontend Patterns
- **No global state library**: All state via `useState` at page level
- **API centralization**: All backend calls in `lib/api.ts`
- **Dual progress tracking**: WebSocket (real-time stages) + REST polling (2s fallback)
- **Immediate final poll**: WS completed/failed → instant REST poll, no polling delay
- **Canvas interaction**: Konva for corner editing

### Naming Conventions
- **Backend**: snake_case for files/functions/vars, PascalCase for classes
- **Frontend**: PascalCase for component files/types, camelCase otherwise
- **Env vars**: UPPER_SNAKE_CASE with `__` nesting delimiter

### Error Handling
- **Backend routes**: HTTPException(400/404)
- **Core modules**: Return None/empty on failure, log warnings
- **Frontend**: `json<T>()` throws on non-ok; polling ignores transient errors

## KNOWN ISSUES & TECH DEBT

1. ~~**Progress bar non-functional**~~ ✅ **RESOLVED**: Pipeline now calls `on_progress` callback at each stage; bridged to WebSocket via `run_coroutine_threadsafe`.
2. **In-memory task storage**: `_tasks` and `_pending_images` are dict-based. No persistence across restarts.
3. **Heatmap neighbor bleed**: CRAFT heatmap activations from adjacent characters can bleed into blank cells at their edges. Analyzed but not yet fixed (solution: use center 60% of ink_crop for heatmap analysis in `_is_blank_cell`).
4. **No CI/CD**: No GitHub Actions or automation pipeline.
5. **Docker unverified**: docker-compose.yml written but not tested on clean environment.
6. **CORS wide open**: `allow_origins=["*"]`
7. **Type sync manual**: `frontend/src/lib/types.ts` must be manually kept in sync with `backend/api/schemas.py`.
8. **Default cuda_device mismatch**: `settings.py` defaults to `cuda:7` (dev machine), override via env var.

## RECIPES

### Recipe: Add a new VLM backend (e.g., Anthropic Claude)
1. Edit `services/vlm_service.py`: add strategy method, update `from_settings()` factory
2. Edit `config/settings.py` → `VLMSettings`: add API key + model name fields
3. Update `.env.example` and `VLMSettings.backend` Literal type

### Recipe: Add a new API endpoint
1. Create `backend/api/routes/myroute.py` with `router = APIRouter(prefix="/myroute")`
2. Add schemas to `schemas.py`
3. Register in `api/app.py`: `app.include_router(myroute.router, prefix=prefix)`

### Recipe: Add a new OCR pipeline stage
1. Create module in `backend/core/`
2. Import and call in `ocr_pipeline.py` → `run()`, add `on_progress(stage_name, value)` call
3. Add settings to `OCRPipelineSettings` if configurable

### Recipe: Fix heatmap neighbor bleed in blank cell detection
In `core/text_reformat.py`, modify `_is_blank_cell()` to use center crop:
```python
_HEAT_CENTER_MARGIN = 0.20  # trim 20% from each edge

def _center_heat_stats(ink_crop, active_threshold):
    if ink_crop is None or ink_crop.size == 0:
        return 0.0, 0.0, 0.0
    h, w = ink_crop.shape[:2]
    my, mx = max(1, int(h * _HEAT_CENTER_MARGIN)), max(1, int(w * _HEAT_CENTER_MARGIN))
    center = ink_crop[my:max(my+1, h-my), mx:max(mx+1, w-mx)]
    return _heat_stats(center, active_threshold)
```
Then replace `_heat_stats(ink_crop, ...)` with `_center_heat_stats(ink_crop, ...)` in `_is_blank_cell`.
